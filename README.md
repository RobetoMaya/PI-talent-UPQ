TalentUPQ es una aplicación web de vanguardia diseñada para optimizar la vinculación laboral entre estudiantes, egresados de la Universidad Politécnica de Querétaro (UPQ) y el sector productivo. El sistema centraliza la gestión de vacantes, perfiles profesionales y procesos de reclutamiento bajo un entorno seguro y validado.

## 🎯 Alcance del Sistema

### Objetivo General
Desarrollar una bolsa de trabajo universitaria integral que conecte el talento académico con ofertas laborales reales mediante herramientas personalizadas.

### Funciones Principales
* **Gestión de Perfiles:** Creación de perfiles profesionales con carga de CV en formato PDF.
* **Módulo de Empresas:** Publicación y administración de vacantes e interacción con candidatos.
* **Panel Administrativo:** Validación proactiva de registros y supervisión de seguridad de las vacantes.
* **Sistema Inteligente:** Integración de IA para sugerencias de vacantes basadas en habilidades técnicas.
* **Notificaciones:** Alertas automatizadas vía correo electrónico y plataforma.

## 🛠️ Especificaciones Técnicas

* **Arquitectura:** Full Stack (Frontend adaptativo y Backend escalable).
* **Infraestructura:** Amazon Web Services (AWS) para garantizar alta disponibilidad.
* **Base de Datos:** SQL Server para gestión robusta de información relacional.
* **Seguridad:** Cifrado de datos en tránsito y reposo, cumplimiento con la LFPDPPP (Protección de Datos Personales).
* **Accesibilidad:** Cumplimiento de pautas WCAG 2.1 nivel AA.

## 📋 Ingeniería de Requisitos (Resumen MoSCoW)

### Must Have (Obligatorios)
* Registro diferenciado de roles (Estudiante, Empresa, Admin).
* Validación de vacantes por la institución (Filtro antifraude).
* Autenticación segura y protocolos HTTPS.
* Cumplimiento legal de privacidad de datos.

### Should Have (Deseables)
* Notificaciones automáticas.
* Dashboard interactivo para toma de decisiones.
* Exportación de reportes en formatos PDF/Excel.

## 📅 Cronograma de Implementación

El proyecto se estructura en un ciclo de **12 meses**, con una liberación de la primera versión funcional (MVP) al finalizar el cuarto mes.

1. **Mes 1-3:** Desarrollo del núcleo y funciones principales.
2. **Mes 3:** Pruebas internas y corrección de errores.
3. **Mes 4:** Lanzamiento de versión funcional.
4. **Mes 5-12:** Refinamiento, optimización y escala de usuarios.
